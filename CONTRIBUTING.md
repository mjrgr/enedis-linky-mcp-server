# Contributing to Enedis Linky MCP Server

Thank you for your interest in contributing! We welcome all kinds of contributions, including bug reports,
 feature requests, documentation improvements, and code changes.

## How to Contribute

1. **Fork the repository**
2. **Create a new branch** for your feature or fix
3. **Make your changes** and add tests if needed
4. **Run tests and lint** to ensure code quality
5. **Open a pull request** with a clear description of your changes

## Code Style

- Follow Go best practices and idioms
- Use `go fmt` to format your code
- Write clear, concise commit messages

## Reporting Issues

- Use [GitHub Issues](https://github.com/mjrgr/enedis-linky-mcp-server/issues) for bugs and feature requests
- Provide as much detail as possible (logs, steps to reproduce, environment)

## Pull Request Guidelines

- Reference related issues in your PR description (e.g., "Closes #42")
- Keep PRs focused and minimal; open separate PRs for unrelated changes
- Ensure all tests pass before requesting review

## Questions?

- Use [Discussions](https://github.com/mjrgr/enedis-linky-mcp-server/discussions) for Q&A and ideas
- For sensitive issues, email the maintainers

Thank you for helping make this project better!
